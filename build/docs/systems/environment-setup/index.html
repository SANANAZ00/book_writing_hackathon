Setting Up the Development Environment

Hero Insight

Transform your development workflow with a properly configured environment that enables efficient AI-powered documentation development.

Purpose of This Chapter

You will set up a complete development environment for building AI-powered documentation systems, including Docusaurus, FastAPI, Qdrant integration, and all necessary tools for the project.

Deep Breakdown

Prerequisites

Before starting, ensure you have:

System Requirements

  • Operating System: Windows, macOS, or Linux
  • Node.js: Version 18.x or higher for Docusaurus
  • Python: Version 3.9 or higher for FastAPI backend
  • Git: For version control and repository management
  • Package Managers: npm/yarn for frontend, pip for backend

Development Tools

  • Code Editor: VS Code with Python and JavaScript extensions
  • Terminal: Command line interface for project management
  • Browser: Modern browser for testing and development
  • API Client: Postman or similar for API testing

Installing Node.js and npm

For Docusaurus development:

# Check current versions
node --version
npm --version

# Install Node.js (includes npm)
# Visit nodejs.org and download LTS version
# Or use a version manager like nvm

Installing Python and pip

For FastAPI backend development:

# Check current versions
python --version
pip --version

# Install Python 3.9+ from python.org
# Or use a version manager like pyenv

Setting Up Docusaurus Frontend

Create the Docusaurus documentation site:

# Navigate to project directory
cd ai-book-project

# Create frontend directory
mkdir frontend && cd frontend

# Initialize Docusaurus project
npx create-docusaurus@latest website classic

# Install additional dependencies
npm install @docusaurus/module-type-aliases @docusaurus/types

Setting Up FastAPI Backend

Create the FastAPI backend services:

# Navigate back and create backend directory
cd ..
mkdir backend && cd backend

# Create virtual environment
python -m venv venv

# Activate virtual environment
# On Windows:
venv\Scripts\activate
# On macOS/Linux:
source venv/bin/activate

# Create requirements.txt
echo "fastapi==0.104.1
uvicorn==0.24.0
pydantic==2.5.0
python-dotenv==1.0.0
qdrant-client==1.7.0
openai==1.3.5
tiktoken==0.5.2" > requirements.txt

# Install dependencies
pip install -r requirements.txt

Environment Configuration

Set up environment variables for API keys and configuration:

# Create .env file in backend directory
echo "OPENAI_API_KEY=your_openai_api_key_here
QDRANT_URL=your_qdrant_cluster_url
QDRANT_API_KEY=your_qdrant_api_key
ENVIRONMENT=development" > .env

Basic FastAPI Application

Create the main FastAPI application:

backend/app/main.py:

from fastapi import FastAPI
from fastapi.middleware.cors import CORSMiddleware
from dotenv import load_dotenv
import os

# Load environment variables
load_dotenv()

app = FastAPI(
    title="AI Documentation Backend",
    description="Backend services for AI-powered documentation system",
    version="1.0.0"
)

# Configure CORS for local development
app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],  # In production, specify exact origins
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)

@app.get("/")
async def root():
    return {"message": "AI Documentation Backend API"}

@app.get("/health")
async def health_check():
    return {"status": "healthy"}

if __name__ == "__main__":
    import uvicorn
    uvicorn.run(app, host="0.0.0.0", port=8000)

Development Workflow Setup

Configure your development workflow:

Git Configuration

# Initialize git repository
git init

# Create .gitignore
echo "node_modules/
venv/
.env
__pycache__/
*.pyc
.DS_Store
dist/
build/" > .gitignore

# Add and commit initial files
git add .
git commit -m "Initial project setup"

Package.json for Root

Create a root package.json for easier management:

package.json:

{
  "name": "ai-book-project",
  "version": "1.0.0",
  "description": "AI-powered documentation system",
  "scripts": {
    "frontend:dev": "cd frontend && npm run start",
    "frontend:build": "cd frontend && npm run build",
    "backend:dev": "cd backend && python -m uvicorn app.main:app --reload --port 8000",
    "backend:prod": "cd backend && python -m uvicorn app.main:app --host 0.0.0.0 --port 8000",
    "setup": "npm install && cd frontend && npm install && cd ../backend && pip install -r requirements.txt"
  },
  "devDependencies": {
    "concurrently": "^8.2.0"
  }
}

Mistake: Not using virtual environments for Python

Fix: Always create and activate a virtual environment before installing Python packages.

Mistake: Exposing API keys in version control

Fix: Use .env files and ensure they're included in .gitignore.

Mistake: Inconsistent port configurations

Fix: Document and standardize port assignments for different services.

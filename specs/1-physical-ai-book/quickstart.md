# Quickstart Guide: Physical AI & Humanoid Robotics Book Development

## Overview
This guide provides the essential steps to set up your development environment for creating the Physical AI & Humanoid Robotics book using Docusaurus v2.

## Prerequisites
- Node.js (version 18 or higher)
- npm or yarn package manager
- Git version control
- Basic knowledge of Markdown and React
- Access to ROS 2, Gazebo, Unity, and NVIDIA Isaac environments

## Environment Setup

### 1. Install Node.js and Package Manager
```bash
# Check if Node.js is installed
node --version

# If not installed, download from https://nodejs.org/
# Or use a version manager like nvm
nvm install node
nvm use node
```

### 2. Install Docusaurus CLI
```bash
npm init docusaurus@latest my-book classic
cd my-book
```

### 3. Project Structure Setup
```
my-book/
├── blog/                 # Optional blog section
├── docs/                 # Documentation files (your book content goes here)
│   ├── intro.md
│   ├── module-1-ros2.mdx
│   ├── module-2-simulation.mdx
│   └── ...
├── src/
│   ├── components/       # Custom React components
│   └── pages/            # Static pages
├── static/               # Static assets (images, etc.)
├── docusaurus.config.js  # Configuration file
├── package.json
└── sidebars.js          # Navigation structure
```

### Exercise 1: Creating Your First Node

Create a simple ROS 2 node that publishes a "Hello, Robot!" message...

```python
import rclpy
from rclpy.node import Node
from std_msgs.msg import String

class MinimalPublisher(Node):
    def __init__(self):
        super().__init__('minimal_publisher')
        self.publisher_ = self.create_publisher(String, 'topic', 10)
        timer_period = 0.5  # seconds
        self.timer = self.create_timer(timer_period, self.timer_callback)
        self.i = 0

    def timer_callback(self):
        msg = String()
        msg.data = f'Hello Robot: {self.i}'
        self.publisher_.publish(msg)
        self.get_logger().info(f'Publishing: "{msg.data}"')
        self.i += 1

def main(args=None):
    rclpy.init(args=args)
    minimal_publisher = MinimalPublisher()
    rclpy.spin(minimal_publisher)
    minimal_publisher.destroy_node()
    rclpy.shutdown()

if __name__ == '__main__':
    main()
```

## Development Workflow

### 1. Start the development server
```bash
npm start
```
This command starts a local development server and opens your book in the browser. Most changes are reflected live without having to restart the server.

### 2. Build for production
```bash
npm run build
```
This command generates static content into the `build/` directory and can be served using any static hosting service.

### 3. Deploy to GitHub Pages
```bash
GIT_USER=<Your GitHub username> npm run deploy
```

## Best Practices for Book Content

### 1. Frontmatter Standards
Every MDX file should include:
```mdx
---
title: "Descriptive Title"
sidebar_position: X  // Sequential numbering
description: "Brief description for SEO"
tags: ["robotics", "ros2", "humanoid"]  // Relevant tags
---
```

### 2. Content Structure
- Use H1 for the main title (automatically generated from frontmatter)
- Use H2 for major sections
- Use H3 for subsections
- Include clear learning objectives
- Add practical exercises with solutions
- Provide code examples with explanations

### 3. Code Example Formatting
```mdx
import CodeBlock from '@theme/CodeBlock';

## Troubleshooting

### 1. Frontmatter Issues
- Ensure all frontmatter values are properly quoted if they contain special characters
- Check that sidebar_position numbers are sequential and unique

### 2. Link Issues
- Use relative paths for internal links: `[link text](./other-page.mdx)`
- Use absolute paths for static assets: `![alt text](/img/image.png)`

### 3. Component Issues
- Ensure all custom components are properly imported
- Check that MDX syntax is valid
